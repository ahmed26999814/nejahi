# MauriResults Mobile Release Policy

هذه قاعدة دائمة لكل إصدار جديد من تطبيق MauriResults:

1. زيادة رقم الإصدار وAndroid version code وعدم إعادة استخدام رقم إصدار سابق.
2. تشغيل Flutter analyze والاختبارات وبناء Release APK قبل النشر.
3. نشر APK الجديد فورًا في صفحة `/Apk` بالموقع الرسمي.
4. حذف ملف النسخة القديمة أو تحويل جميع روابطها إلى أحدث APK.
5. تحديث `/apk/version.json` بالإصدار الجديد و`minimumSupportedVersion`.
6. استخدام `forceUpdate: true` عند إيقاف النسخ القديمة.
7. إظهار شاشة واضحة داخل النسخ القديمة تقول إن النسخة توقفت، مع زر مباشر لتحميل النسخة الجديدة.
8. يجب أن يبقى رابط `/api/apk-download` موجّهًا دائمًا إلى أحدث ملف APK فقط.

الأولوية: لا نشر لإصدار جديد دون تحديث الموقع وإغلاق النسخة القديمة وإظهار زر التحديث للمستخدمين السابقين.
